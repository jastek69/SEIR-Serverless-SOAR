## 2。考えられる説明を可能性別にランク付け

### ディメンション A: トークンが発行されるのに使われない理由

#### 1. ✅ 良性のユーザーによる放棄 (確率:高 — 45% 未満)
**説明:** ユーザー認証 (ログインリンクのクリック、SSO フローの開始など) は完了したが、何らかのアクションを完了する前にセッションを放棄した (ブラウザタブを閉じた、気が散った、ログイン後に UI エラーが発生した)。

**サポート指標:**
-以前に失敗した認証試行なし
-1 回の認証イベント (後続の API 呼び出しなし)
-ブラウザベースのアクセスと一貫性のあるユーザーエージェント

**リスクレベル:** 低。悪意のある意図はないが、トークンは有効なままであり、認証情報が失われています。

---

#### 2. ✅ 自動スクリプト/ボット認証プローブ (可能性:中～高 — 25% まで)
**説明:** 自動攻撃者 (資格情報収集ツール、偵察ボット) が、有効な認証情報 (フィッシング、データ漏えい、またはパスワードスプレーによって入手された可能性がある) を使用して正常に認証されたが、以下のいずれかの理由でトークンの使用を開始しなかった。
-認証後にスクリプトが失敗した
-攻撃者は、すぐに悪用されることを意図せずに認証情報の有効性をテストしていました。
-トークンは後で使用するために収集された (トークンの流出)

**サポート指標:**
-通常とは異なるIP、ASN、または位置情報からの認証
-非標準のユーザーエージェント文字列
-通常とは異なる時間帯の認証
-以前に失敗した試みがない (ブルートフォースではなく、有効な認証情報が提示される)

**攻撃経路:**
```
クレデンシャルスタッフィング DB → 有効なクレデンシャルが見つかりました → 認証エンドポイントヒット →
JWT 発行 → トークンが C2 に流出 → 悪用の遅延
```

**リスクレベル:** 高。このようなシナリオの場合、トークンは敵の手に渡るライブ認証情報となります。

---
#### 3. ✅ 統合/サービスアカウントの設定ミス (可能性:中 — ~ 15%)
**説明:** サービスアカウントまたは API 統合は正常に認証されたが、設定ミス、ネットワークエラー、またはダウンストリームサービスの障害により続行できませんでした。トークンは発行されたが、利用側サービスが API 呼び出しを行わなかった。

**サポート指標:**
-静的 IP (サーバー/クラウドプロバイダーの範囲) からの認証
-サービスアカウントのユーザー名パターン
-アプリケーションログ内の相関インフラストラクチャーエラー
-複数の時間枠にわたって繰り返されるパターン

**リスクレベル:** 低～中。悪質ではないが、インテグレーションが壊れているため、再試行によって不要なトークンチャーンが発生している可能性がある。

---

#### 4. ⚠️ XSS またはクライアント側攻撃によるトークンの流出 (発生率:低中程度 — ~ 10%)
**説明:** ユーザーは合法的に認証されました。トークンはブラウザに発行されましたが、ユーザーが認証されたリクエストを行う前に、XSS ペイロードを介して直ちに XSS ペイロードを介して流出しました。トークンが使用前に盗まれたため、正規のユーザーセッションは「未使用」のように見えました。

**攻撃経路:**
```
ユーザーが悪意のある、または侵害されたページにアクセスする →
XSS ペイロードは認証コールバック時に起動します →
ローカルストレージ/クッキーから抽出された JWT →
攻撃者が制御するエンドポイントに投稿されたトークン →
正規のセッションがアイドル状態に見える
```

**サポート指標:**
-ユーザーが最近、外部またはユーザー生成コンテンツページにアクセスした
-ほぼ同時期に CSP 違反が記録された
-予期しないドメインへの送信ネットワークログにトークンが表示される

**リスクレベル:** 高。トークンは完全なユーザー権限で敵の手に渡ります。

---

#### 5. ⚠️ 内部脅威/認証情報の共有 (発生可能性:低い — ~ 5%)
**説明:** ユーザーが認証され、トークンを不正な第三者に共有または転送しました。トークンが別の場所で使用されている間に、元のセッションが中止されました (ただし、「使用したことがない」ということは、二次使用も個別に追跡されない限り、その可能性は低いことを示唆しています)。

**リスクレベル:** ユーザーの権限レベルにもよりますが、中～高。

---

### ディメンション B: ディメンション B: 検出器がスキャンされたレコードを 0 件返した理由

#### 1. ✅ データソース接続障害 (可能性:高 — 40% 未満)
ディテクターはトークンストア (Redis、DynamoDB、RDS など) に接続できませんでした。接続タイムアウト、IAM 権限エラー、またはエンドポイントの設定ミスです。

#### 2. ✅ トークン・ストアが空/アクティブなセッションがない (可能性:中 — ~ 30%)
スキャン時には、システムにはアクティブなトークンがまったくありません。トラフィックの少ない環境、セッションをフラッシュしたデプロイメント後、または営業時間外でも可能です。

#### 3. ✅ クエリ/フィルターロジックのバグ (発生率:中 — ~ 20%)
ディテクタのクエリが不適切にフィルタリングされ (タイムウィンドウの比較が間違っている、タイムゾーンの不一致、インデックスが間違っているなど)、レコードが存在していても結果が返されない。

#### 4. ⚠️ ディテクタのデプロイメント/設定エラー (発生可能性:低～中 — ~ 10%)
Lambda 関数、コンテナ、またはスクリプトの設定が間違っている — 環境変数、リージョン、テーブル/インデックス名が間違っている。
