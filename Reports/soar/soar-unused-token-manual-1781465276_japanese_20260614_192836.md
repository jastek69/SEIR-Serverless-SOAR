## 1。重要度評価

**重要度:低～中 (コンテキスト) **
**信頼度:中程度** (スキャンコンテキストでは検出結果がゼロのため、シグナルが限定的) *

### 正当化

表面的には、認証が成功してから未使用の JWT トークンが使用されても問題がないように見えます。ユーザーがセッションを放棄したり、ブラウザを閉じたり、接続の問題が日常的に発生したりします。ただし、次のような特定の条件下では、このパターンが分析的に重要になります。

| 要因 | リスクの寄与 |
|---|---|
| 発行されたがまだ使用されていないトークン | 悪用されずに認証情報を収集できる可能性 |
| 15 分以内に発行 (5 分のしきい値を超える) | 意図的または自動的な発行を提案 |
| 検出器スキャンで何も検出されない | 検出器に隙間がある可能性がある。手動トリガーでは精査が必要 |
| オペレーターによる手動トリガー | 既存の疑いまたは日常的な監査を意味する |

>**重要なニュアンス:** イベントコンテキスト JSON には「スキャン済み:0」と「結果合計:0」が表示されます。これは、ディテクタが**トークンをまったくスキャンしなかった**ことを示すもので、それ自体が異常です。ディテクタは動作しているが処理するレコードがゼロの場合は、リスクがないということではなく、**パイプライン障害、設定ミス、またはデータ取り込みギャップ**を示唆しています。

重大度が「低」から「中」に上がる主な理由は次のとおりです。
-このレビューサイクルでは、検出インフラストラクチャが機能していないようです。
-オペレーターが手動でレビューを開始せざるを得ないと感じたため、事前の懸念を示唆している
-トークンの監視に盲点があると、悪用されやすい検出ギャップが生じる

## 2。考えられる説明を可能性別にランク付け

### ランク 1 — 良性のユーザーによる放棄* (~ 45% の確率) *
ユーザーは認証され、トークンを受け取り、その後:
-アクションを完了する前にブラウザタブを閉じた
-ネットワーク接続が切断されました
-中断され、ウィンドウ内に戻らなかった
-クライアント側でアプリケーションがクラッシュした

**問題ないにもかかわらず重要な理由:** 合法的に大規模な放棄が行われたとしても、傍受された場合に攻撃対象となる監視されていない有効なトークンのプールができてしまいます。

---

### ランク 2 — ディテクタ/パイプラインの障害* (~ 30% の確率) *
`scanned: 0`フィールドは、このイベントにおける運用上最も重要なデータポイントです。考えられる原因:
-スキーマの変更またはインデックス障害により、トークンストアクエリが空を返す
-タイムウィンドウフィルターの設定が誤っている（UTC オフセットエラーが原因で、レコードが範囲内に収まらないなど）
-メッセージキューまたはログパイプラインのバックログにより取り込みが遅れる
-ディテクタージョブ自体が黙って失敗し、成功を報告して結果が空になる

**攻撃経路への影響:** ディテクターが壊れた場合、フィッシング、MITM、またはクレデンシャルスタッフィングによってトークンを入手した攻撃者は、アラートを発生させずに (トークンの有効期限が切れるまで無期限) トークンを保持できます。

---

### ランク 3 — クレデンシャル・ハーベスティング/トークン・セフト・プローブ * (~ 15% の確率) *
攻撃者は、盗んだ認証情報やブルートフォース認証情報を利用して、以下の目的で認証を行う可能性があります。
-ユーザーベースの検出を行わずに、認証情報が有効であることを検証する
-有効な JWT を収集して後で使用できるようにする (監視期間外で)
-偵察の一環として認証エンドポイントをテスト
-認証情報の検証を大規模に自動化 (認証情報の詰め込み検証フェーズ)

**攻撃経路の詳細:**
```
攻撃者が認証情報リストを取得
 ↓
/auth エンドポイントに対して認証を行います。
 ↓
JWT の発行とキャプチャ (インターセプトプロキシ経由など)
 ↓
トークンは後で使用または販売できるように保管されています
 ↓
ダウンストリーム API 呼び出しなし → 動作検出を回避
 ↓
数時間/数日後に別の IP/デバイスから使用されたトークン
```

このパターンは、認証イベントを悪用イベントから切り離すことで、「不可能な移動」や「異常な API 使用」の検出を回避するように特別に設計されています。

---
### ランク 4 — 自動テストまたは CI/CD パイプライン* (~ 7% 確率) *
-認証はされるがユーザーフロー全体は完了しない統合テスト
-後続の API 呼び出しなしで認証トークンを生成するロードテストツール
-ステージングとプロダクションの重複シナリオにおけるサービスアカウントの設定ミス

---

### ランク 5 — 内部脅威または特権列挙* (~ 3% の確率) *
内部アカウントまたは侵害を受けた内部アカウントが、以下を認証している。
-計画された悪質なアクションが実行される前に、アクセスがまだ存在することを確認する
-アカウントにフラグが立てられているか、制限されているかをテストする
-トークンを生成して外部に流出させる

---
